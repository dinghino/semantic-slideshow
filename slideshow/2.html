how do I test it?

The app works by loading html pages that are found inside a folder, like slideshow that you've seen in the previous page. You also need to specify the number of slides that you want to load.

Ok... but how do I create slides?

Slides are just html files with just a div container inside.

Inside that div you can do whatever you want. You can use semantic-ui to style the content too, which is awesome. Remember that the slides get flexbox from css, so you can use and experiment with that.

Also remember that the slides can be in any folder you want to (so you can have multiple slideshows to load) but must have predetermined name, as in numeric, starting from 0.html, without missing any.

And what about events and script?
Since the slides are just html, you can add a script tag anywhere and the content will load and execute. The suggestion is to use the first slide to load everything, or split each slide functionalities for each slide, if there is lots to do...Since the pages will be loaded together, though, this is just a design preference.
hooks and app events
You can also access and utilize app events and methods while writing down your slides, but there is a catch. If you want to access hooks you have to create an object inside a script tag that needs to be called _slidesEvents. inside that you have to nest an object for every slide you want to access the hooks, and define the callbacks functions for those events.
